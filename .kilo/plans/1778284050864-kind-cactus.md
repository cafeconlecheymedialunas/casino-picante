# Plan: Remove line selector from sale form; require global line filter first

## Summary

Remove the inline "Línea" selector from the sale creation/edit modal. Instead, the user **must** select a line from the global toolbar filter (`lineFilter`) before opening the modal. The sale form will inherit the line from the global filter and only show agent/client/platform selectors (loaded dynamically based on that line).

Also fix the `line_clients.client_id` → `line_clients.user_id` column bug.

---

## Changes

### 1. `app/Livewire/Ventas.php`

#### 1a. Fix column bug (line 268)
- Change `line_clients.client_id` → `line_clients.user_id`

#### 1b. `openCreateModal()` — require lineFilter to be set
- If `lineFilter === 'all'`, do not open the modal (return early or do nothing).
- Remove the existing logic that pre-selects the first agent — the form will just inherit the line from `lineFilter`.
- Set `$this->saleLineId = $this->lineFilter` before opening the modal.

#### 1c. `openEditModal()` — no changes needed
- Editing an existing sale already has a line_id on the sale record, so it works regardless of `lineFilter`.

#### 1d. `closeModal()` — reset saleLineId
- Already calls `resetSaleForm()` which clears `saleLineId`. No change needed.

#### 1e. `resetSaleForm()` — no change needed
- Already resets `saleLineId` to `''`.

#### 1f. `saveSale()` — validation
- The `saleLineId` validation (`required|integer|exists:lines,id`) still works since `saleLineId` will be set from the hidden field.

### 2. `resources/views/livewire/ventas.blade.php`

#### 2a. "Cargar venta" button — disable when no line selected
- Add a conditional: if `lineFilter === 'all'`, either hide the button or disable it with a tooltip/title explaining the user must select a line first.
- Simplest approach: wrap in `@if($lineFilter !== 'all')` so the button only appears when a line is selected.

#### 2b. Modal — remove line selector entirely
- Remove the entire `@if($lineFilter !== 'all')` / `@else` block (lines 148-167) that shows either the hidden field+display or the dropdown.
- Replace with a simple hidden input that always sets `saleLineId` from `lineFilter`:
  ```html
  <input type="hidden" wire:model="saleLineId" value="{{ $lineFilter }}">
  ```
- Optionally show a small read-only display of the selected line name for clarity (similar to the toolbar display).

#### 2c. Form layout adjustment
- Since the line field is now hidden, the form grid may need slight adjustment. The agent/client/platform selectors remain as-is.

---

## Behavior After Changes

1. User lands on `/ventas` — sees the global line selector set to "Todas las lineas"
2. "Carga venta" button is hidden/disabled until user picks a specific line
3. User selects a line from the global filter → "Carga venta" button appears
4. User clicks "Carga venta" → modal opens with the line pre-set (not user-changeable in the modal)
5. Agent, client, platform dropdowns load dynamically based on the selected line
6. User fills the rest of the form and saves
