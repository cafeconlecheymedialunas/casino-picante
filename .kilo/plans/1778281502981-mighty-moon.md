# Plan: Remove Sales Adding Functionality and Use Calculated Values

## Current State Analysis

The sales module consists of:
1. **Main Sales Page** (`resources/views/livewire/ventas.blade.php`) - Full sales management with create/edit/delete functionality
2. **Lines Management Page** (`resources/views/livewire/lineas.blade.php`) - Contains a "quick sales" modal for adding/editing sales per line
3. **Line Detail Page** (`resources/views/livewire/line-detail.blade.php`) - Shows sales statistics/KPIs, currently using stored manual values

## Issues Identified

The user wants to:
1. Remove the ability to add sales "desde el detalle de la linea" (from the line detail/line management)
2. Use calculated sales values instead of stored manual values in the line detail

Currently:
- Sales can be added from the lines management page via the "Ventas" button on line cards
- Line detail shows sales stats using stored fields like `$line->ventas_mes_actual` (being removed by migration)
- The LineDetail component has calculation methods but they're not being used for the basic stats display

## Proposed Changes

### 1. Remove Sales Button from Line Cards
**File:** `resources/views/livewire/partials/line-card.blade.php`
- Remove the "Ventas" button (line 128-131) from the line card footer actions

### 2. Remove Sales Modal from Lines Management Page
**File:** `resources/views/livewire/lineas.blade.php`
- Remove the entire sales modal section (lines 854-922)
- This includes the modal overlay, form, and sales table display

### 3. Remove Sales Modal Logic from Lineas Component
**File:** `app/Livewire/Lineas.php`
- Remove sales-related properties: `$showSalesModal`, `$salesLine`, `$salePlatformId`, `$saleDate`, `$saleDescripcion`, `$saleMontoFichas`, `$editingSaleId`
- Remove sales modal methods: `openSalesModal()`, `closeSalesModal()`, `saveSale()`, `deleteSale()`
- Remove sales data loading in `render()` method

### 4. Update Line Detail to Use Calculated Values
**File:** `resources/views/livewire/line-detail.blade.php`
- The `@php` block already calculates values correctly, but some display sections still use deprecated stored fields
- Update the sales KPIs section to use calculated variables:
  - Lines 391-393: Replace `$line->mejor_plataforma` and `$line->mejor_plataforma_total` with `$bestPlatform` data (already correct)
  - Line 401: Replace `${{ number_format($line->ventas_mes_actual ?? 0, 2) }}` with `${{ number_format($totalThisMonth, 2) }}`
  - Lines 406-407: Update the "Ganancia Encargado" card to display data from `$encargadoEarnings` array instead of `$line->porcentaje_encargado` and `$line->ganancia_encargado`
- The edit section (line 288) uses `wire:model="line.porcentaje_encargado"` which is being removed. Replace this inline input with a button that calls `openPercentageModal({{ $la->agent->id }})` to edit the specific agent's percentage

### 5. Remove Manual Stats Fields from Line Model
**File:** `app/Models/Line.php`
- Remove manual sales fields from fillable array if they exist
- Ensure the model doesn't expect these fields to be stored

## Benefits

1. **Simplified UX**: Sales management is centralized in the dedicated sales page only
2. **Reduced Complexity**: Removes duplicate sales entry points and manual data entry
3. **Data Accuracy**: Sales stats are always calculated from actual sales data, not manually entered
4. **Clear Separation**: Line management focuses on line configuration, sales management is separate
5. **Consistent Pattern**: All sales operations go through the main sales page

## Additional Context

The migration `2026_05_08_230000_remove_manual_stats_fields_from_lines.php` removes the following manual sales stats fields from the `lines` table:

**Fields being removed:**
- `mejor_mes` - Best month (text)
- `mejor_mes_total` - Best month total sales (decimal)
- `mejor_plataforma` - Best platform (text)
- `mejor_plataforma_total` - Best platform total sales (decimal)
- `ventas_mes_actual` - Current month sales (decimal)
- `ventas_mes_pasado` - Past month sales (decimal)
- `ventas_mes_antiguo` - Previous month sales (decimal)
- `ganancia_encargado` - Encargado earnings (decimal)
- `porcentaje_encargado` - Encargado percentage (decimal)

These fields contained manually entered sales statistics. The plan ensures the application uses calculated values from actual sales data instead of relying on these deprecated stored fields.

## Testing Requirements

After implementation:
1. Verify sales can still be managed through the main sales page
2. Verify line detail page shows calculated sales statistics correctly
3. Verify line management page no longer has sales functionality
4. Test that calculated values match actual sales data
5. Test that removing sales functionality doesn't break any other features
6. Run the migration to remove manual stats fields and verify no data loss

## Rollback Plan

If issues arise, the changes can be reverted by:
1. Restoring the removed sales modal section in `lineas.blade.php`
2. Restoring the sales button in `line-card.blade.php`
3. Restoring the sales modal logic in `Lineas.php`
